# Генеалогическое древо

Клиентское веб-приложение для создания, просмотра и редактирования семейных деревьев. Все данные хранятся **локально** в браузере и в файлах проекта на компьютере пользователя — сервер не требуется.

## Возможности

### Проекты и файлы

- Создание пустого проекта с нуля
- Открытие сохранённого проекта (`.drevo` или `.zip` с тем же содержимым)
- Импорт GEDCOM (Familio, FamyTale и другие программы)
- Экспорт проекта в GEDCOM
- Сохранение и «Сохранить как» в файл `.drevo`
- Список недавно открытых проектов (локально в браузере)

### Древо

- Интерактивная SVG-визуализация с центром на выбранной персоне или семье
- Настройка глубины поколений вверх/вниз, боковых ветвей
- Режимы размера карточек: одинаковый или уменьшаемый
- Темы оформления: «Чёткие линии» и «Лес»
- Настройка полей на карточках: ФИО, прозвище, годы жизни, возраст, место, фото, годы брака
- Поиск по ФИО, годам и местам
- Ручное расположение карточек (режим редактирования)
- Экспорт листа древа в PNG, JPEG или PDF

### Личное дело (досье)

- ФИО, пол, даты и места рождения/смерти, биография
- Поддержка **юлианского календаря** (даты с суффиксом « ст.»)
- Аватар и медиаархив: фото, видео, аудио, документы (PDF и др.)
- **Управление связями** при редактировании карточки персоны (кнопка «Редактировать» в досье):
  - родители, партнёры, дети;
  - создание новой персоны или привязка существующей через поиск;
  - удаление связи без удаления персоны из проекта;
  - несколько союзов (браков), дети без партнёра, даты брака

### Режимы

| Режим | URL | Описание |
|-------|-----|----------|
| Просмотр | `/tree` | Только чтение, без изменения данных |
| Редактирование | `/edit` | Добавление персон, редактирование досье (поля и связи), медиа, сохранение |

## Требования

| Компонент | Версия / рекомендация |
|-----------|------------------------|
| **Node.js** | 20.x или новее (LTS) |
| **npm** | 10.x или новее (идёт вместе с Node.js) |
| **Браузер** | Chrome 90+, Edge 90+, Firefox 90+ (современный Chromium предпочтителен) |
| **ОС** | Windows 10+, macOS 12+, Linux |
| **Интернет** | Нужен только для первой установки зависимостей (`npm install`) |

Проверить версии в терминале:

```bash
node -v
npm -v
```

Ожидаемый результат, например: `v22.x.x` и `10.x.x`.

## Демо на GitHub Pages

Онлайн-версия (без установки Node.js): **[https://gregimuri.github.io/Family_Tree/](https://gregimuri.github.io/Family_Tree/)**

Деплой выполняется автоматически при push в ветку `main` (workflow `.github/workflows/deploy-pages.yml`).

### Первичная настройка в репозитории GitHub

1. Откройте **Settings → Pages**
2. В **Build and deployment → Source** выберите **GitHub Actions**
3. Запушьте изменения в `main` — после успешного workflow сайт станет доступен по ссылке выше

### Локальная проверка сборки для Pages

```bash
npm run build:pages
npm run preview -- --base /Family_Tree/
```

Откройте адрес из вывода `preview` (обычно `http://localhost:4173/Family_Tree/`).

> На GitHub Pages данные по-прежнему хранятся только в браузере пользователя. File System Access API может быть ограничен — используйте «Сохранить как» для скачивания `.drevo`.

## Установка и запуск на другом компьютере

Ниже — пошаговая инструкция для **тестировщика** или любого, кто получил исходный код и хочет запустить приложение локально.

### Шаг 1. Установить Node.js (если ещё не установлен)

1. Откройте [https://nodejs.org/](https://nodejs.org/)
2. Скачайте версию **LTS** (рекомендуется)
3. Установите с настройками по умолчанию (галочка «Add to PATH» должна быть включена)
4. Перезапустите терминал и проверьте:

```bash
node -v
npm -v
```

**Windows:** можно использовать «Терминал» (Windows Terminal), PowerShell или cmd.  
**macOS / Linux:** Terminal или любой эмулятор терминала.

### Шаг 2. Получить исходный код проекта

Выберите один из способов:

**Вариант A — через Git (если есть доступ к репозиторию):**

```bash
git clone https://github.com/Gregimuri/Family_Tree
cd Family_Tree
```

**Вариант B — из ZIP-архива:**

1. Распакуйте архив в удобную папку, например `C:\Projects\Family_Tree` или `~/Projects/Family_Tree`
2. Откройте терминал **в этой папке**

В Windows: в Проводнике откройте папку → адресная строка → введите `cmd` или `powershell` → Enter.

Убедитесь, что в папке есть файлы `package.json`, `README.md`, каталог `src/`.

### Шаг 3. Установить зависимости

В корне проекта выполните **один раз**:

```bash
npm install
```

- Команда скачает все библиотеки в папку `node_modules/` (это может занять 1–3 минуты)
- При успехе ошибок в конце не будет; предупреждения (warnings) обычно не критичны

> **Важно:** папку `node_modules` не нужно копировать вручную — она создаётся командой `npm install` на каждом компьютере отдельно.

### Шаг 4. Запустить приложение в режиме разработки

```bash
npm run dev
```

В терминале появится адрес, обычно:

```
  ➜  Local:   http://localhost:5173/
```

1. Откройте этот адрес в браузере (Chrome или Edge предпочтительны)
2. Должен открыться **стартовый экран** с кнопками «Создать проект», «Открыть проект», «Импорт GEDCOM»
3. **Не закрывайте терминал** — пока он работает, приложение доступно в браузере

Остановить сервер: в терминале нажмите `Ctrl + C`.

### Шаг 5 (альтернатива). Запуск production-сборки

Если нужно проверить приложение ближе к «боевой» версии:

```bash
npm run build
npm run preview
```

Откройте адрес из вывода команды (обычно `http://localhost:4173`).

### Частые проблемы при установке

| Проблема | Решение |
|----------|---------|
| `'node' is not recognized` / `'npm' is not recognized` | Node.js не установлен или не добавлен в PATH. Переустановите Node.js LTS и перезапустите терминал |
| `npm install` падает с ошибкой сети | Проверьте интернет, VPN, прокси; повторите `npm install` |
| Порт 5173 занят | Vite предложит другой порт (5174 и т.д.) — откройте указанный в терминале |
| Белый экран в браузере | Откройте консоль разработчика (F12), обновите страницу; убедитесь, что `npm run dev` всё ещё запущен |
| Антивирус блокирует | Разрешите Node.js; папка проекта не содержит исполняемых файлов кроме стандартных npm-скриптов |

## Инструкция для тестировщика

### Минимальный сценарий проверки

1. **Старт** — открыть `http://localhost:5173`, убедиться, что главная страница загружается
2. **Новый проект** — «Создать проект» → ввести название → попасть на древо в режиме редактирования
3. **Добавить персону** — кнопка «Персона» в шапке → заполнить имя → карточка появилась на древе
4. **Досье** — двойной щелчок по карточке (в режиме просмотра) или клик → открывается боковая панель с данными
5. **Связи** — открыть досье → «Редактировать» → блок «Связи»:
   - «+ Создать родителя» / «Привязать…»
   - партнёр и дети в блоке «Союз»
   - удаление связи кнопкой × (персона остаётся в проекте)
   - без «Редактировать» связи только просматриваются
6. **Режим просмотра** — кнопка «Просмотр» в шапке → редактирование недоступно, древо читается
7. **Сохранение** — «Сохранить» / «Сохранить как» → файл `.drevo`
8. **Открытие** — «Главная» → «Открыть проект» → выбрать сохранённый `.drevo`
9. **GEDCOM** — импорт тестового `.ged` → персоны на древе; экспорт через меню экспорта
10. **Экспорт изображения** — кнопка экспорта на древе → PNG/PDF → файл скачивается, карточки видны
11. **Ручная раскладка** — в режиме редактирования кнопка перемещения карточек (иконка «Move») → перетащить карточку → сохранить проект

### Что приложить к отчёту об ошибке

- ОС и версия браузера
- Шаги воспроизведения
- Ожидаемое и фактическое поведение
- Скриншот или запись экрана
- Тестовый файл (`.drevo` или `.ged`), если ошибка связана с данными
- Сообщения из консоли браузера (F12 → Console)

### Автотесты (опционально)

```bash
npm test
```

Должно завершиться с `Tests ... passed`. Падение тестов — повод сообщить разработчику.

## Работа с приложением

### Стартовый экран

| Действие | Описание |
|----------|----------|
| **Создать проект** | Пустое древо, сразу режим редактирования |
| **Открыть проект** | Файл `.drevo` или `.zip` |
| **Импорт GEDCOM** | Загрузка `.ged` / `.gedcom` |
| **Недавние** | Быстрое открытие проектов из локального хранилища браузера |

### Шапка на экране древа

- **Главная** — возврат на стартовый экран
- **Название проекта** — редактируется в режиме редактирования
- **Просмотр / Редактировать** — переключение режимов
- **Сохранить / Сохранить как** — запись файла `.drevo`
- **Персона** — добавить новую персону (только редактирование)

### Панели на древе

- **Поиск** — найти персону и сфокусировать древо
- **Настройки** — поколения, ветви, карточки, тема
- **Масштаб и полноэкранный режим** — правый нижний угол
- **Ручная раскладка** — кнопка с иконкой перемещения (только `/edit`)

### Личное дело

- Открывается по клику на карточку
- Связи редактируются только после нажатия **«Редактировать»** в досье (нужен также режим редактирования проекта `/edit`)
- Для правки полей персоны используется тот же режим «Редактировать» → «Готово»

## Управление на древе

| Действие | Клавиши / мышь |
|----------|----------------|
| Перемещение по полю | Перетаскивание любой кнопкой мыши, стрелки, WASD |
| Масштаб | Ctrl + колёсико, Ctrl + / Ctrl −, кнопки на панели |
| Прокрутка | Колёсико мыши |
| Горизонтальная прокрутка | Shift + колёсико |
| Открыть досье | Клик по карточке (или двойной щелчок в некоторых режимах) |
| Ручное перемещение карточки | Включить режим раскладки → перетащить карточку |

## Формат проекта `.drevo`

Файл `.drevo` — это ZIP-архив со структурой:

```
project.drevo (zip)
├── project.json      # персоны, союзы, связи, настройки отображения, метаданные медиа
└── media/            # фото, видео, аудио, документы
    ├── photo-....jpg
    └── ...
```

Проект можно переименовать, копировать и передавать другому пользователю. Для открытия достаточно выбрать файл через «Открыть проект».

## Ограничения браузера

- Данные проекта и медиа хранятся **локально** (IndexedDB + файлы на диске)
- В Chrome и Edge доступно **прямое сохранение** через File System Access API («Сохранить» перезаписывает тот же файл)
- В Firefox и других браузерах сохранение идёт через **скачивание файла** («Сохранить как»)
- Большие медиафайлы увеличивают размер `.drevo`
- При очистке данных сайта в браузере список «Недавних проектов» может пропасть (сами файлы `.drevo` на диске не удаляются)
- Для экспорта PNG/PDF используется растеризация карточек — при очень больших деревьях экспорт может занять несколько секунд

## Команды для разработки

| Команда | Назначение |
|---------|------------|
| `npm install` | Установить зависимости (первый запуск на машине) |

| `npm run build` | Production-сборка в папку `dist/` |
| `npm run build:pages` | Сборка с базовым путём `/Family_Tree/` для GitHub Pages |
| `npm run preview` | Просмотр собранной версии |
| `npm test` | Запуск автотестов (Vitest) |
| `npm run lint` | Проверка кода ESLint |
